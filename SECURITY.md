# Security Policy

## Supported version

Only the latest commit on the default branch is maintained during the research-preview phase.

## Reporting a vulnerability

Use GitHub's private vulnerability-reporting feature for security-sensitive flaws when it is enabled for this repository. For ordinary, non-sensitive defects, open a GitHub issue with a minimal reproducer and the Python version used.

Do not include credentials, private repository content, patent drafts, personal data, or unpublished evidence in a public issue.

## Security scope

This repository is a research prototype. Its SHA-256 hashes are deterministic fingerprints, not digital signatures. The project does not currently provide trusted timestamping, identity verification, consensus, key custody, hostile-input hardening, or production cryptographic assurance.
