# Public Release Scope

## Included

- Public research descriptions of ECL event certification.
- Dependency-free Python prototypes.
- Deterministic test fixtures and generated-output formats.
- Model limitations and interpretation boundaries.
- Citation and contribution metadata.

## Deliberately excluded

- Provisional and PCT drafts, claim-support matrices, filing receipts, and legal strategy.
- Private proof-of-existence folders and internal certificate chains.
- Absolute local computer paths, machine inventories, and repository recovery records.
- Credentials, tokens, account identifiers, private endpoints, and configuration secrets.
- Trading systems, broker integrations, market data, and financial strategies.
- Device fabrication details, laboratory-sensitive designs, and unpublished materials work.
- Full EMK-UGD vault contents or unpublished cross-repository bridges.
- Raw personal data and unpublished third-party material.

## Publication rule

A file belongs in this repository only when it can be understood, executed, and reviewed without relying on private material. References to excluded systems may describe an interface boundary, but must not reproduce their protected implementation.

## Claim discipline

Public documentation should distinguish among:

1. a definition introduced by the model,
2. a result produced by the included code,
3. a hypothesis requiring external validation, and
4. an independently established result.

The repository currently contains items in categories 1 and 2. It does not present category 2 as category 4 merely because Markdown headings make everything look official.
